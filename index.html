미대출신에 디자이너로 근무하신 이력이 있으신데 어떻게 해서 개발자로 전향하시게 된거죠?

면접관1

혹시 근무하시게 된다면, 디자인 업무까지 모두 맡아서 하실 의향이 있으신가요?

미대를 전공하고 디자이너로 사회생활을 시작했지만 가장 잘하고 좋아하던 능력이 점점 클라이언트 취향에 맞춰 생활의 수단으로 바뀌어가는 과정속에서 디자이너라는 직업에 회의감이 들기 시작할 때 퍼블리셔라는 직업을 알게되었습니다.
이미 갖춰져 있는 디자이너의 능력에서 조금만 노력하면 디자이너가 아닌 다른 직업으로 전향할 수 있겠다라는 부푼 꿈으로 프론트엔드 양성과정을 수료하였고 해당 훈련과정을 이수 완료한 시점에서는 개발 구현 영역까지 욕심이 생겨 추가로 개발자 양성과정까지 수료하면서 개발자로 전향할 수 있게 되었습니다.

보유하고 있는 기술에 디자인 능력도 갖춰 있지만 분명하게 말씀드리자면 디자인 업무는 어떠한 경우에도 맡아서 수행할 생각이 없습니다.
개인 작품 활동이 아닌 상업 디자인은 저와 맞지 않으며, 현재는 본인의 몫은 본인이 직접 수행할 수 있는 개발자가 되는데에 목표를 두고 있기 때문에 다양한 개발 경험과 기술 습득에 전념하고 싶습니다.

면접관2

개발하시면서 기술적으로 어려운 문제를 해결한 경험이 있으신가요? 맞닥뜨렸던 문제가 무었이었는지, 해결을 위해 얼마나 깊이 있게 고민을 했는지 그 과정에 대해 이야기해 주세요.

운영 중이던 하이브리드 앱을 안드로이드 네이티브 앱으로 신규 개발하는 프로젝트를 맡고 있었는데 JSON 형태로 넘어오는 데이터값이 HTML 태그로 나열된 문자열이었어서 XML 데이터 바인딩에 오류가 계속 발생하였고 소스 코드로 HTML을 XML로 변환하는 방법을 찾으려 했지만 미흡한 실력으로 인해 이를 해결하지 못하고 결국 처리 방식을 변경하여 개발을 완료 했습니다.
변경한 개발 처리 방식은 XML에서 데이터 바인딩시에도 오류가 나지 않도록 데이터 값이 문자열로만 출력되는 인터페이스를 별도로 추가 생성하여 연결하는 것으로 해결하였습니다.
가장 최근에 진행했던 프로젝트라서 아직도 HTML을 XML로 변환하는 방법을 찾지 못한 상태이지만 추 후 방법을 모색하여 이를 꼭 해결할 계획입니다.
